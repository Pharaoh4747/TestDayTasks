# Хранение объектов в памяти или Redis

## Плюсы хранения в памяти

1. Больший контроль над способом хранения и представления информации
2. Возможность воздействовать на соотношение быстродействие / объём памяти
3. Меньшая сетевая нагрузка потому что не нужно подключаться к Redis
4. Большая безопасность потому что решение закрытое
5. Отсутствие нужды в приобразовании типов ключей к типам Redis
6. Отсутствие накладных расходов на сериализацию данных
7. Отсутствие сложности масштабирования Redis при больших объёмах данных
8. Возможность реализовать сохранение данных отлично от хранения в памяти
9. Возможность выбрать другой брокер сообщений или обойтись без него
10. Перспективы многопоточности под разные задачи и разных клиентов

## Плюсы хранения в Redis

1. Готовое решение
2. Простая масштабируемость встроенная в Redis
3. Меньшие накладные расходы на разработку потому что не надо разрабатывать Redis
4. Готовые средства для просмотра данных
5. Готовое средство сохранения данных
